## FIX 1: Repair Newsletter Route Registration & Auth (CRITICAL)

### Root Cause
The newsletter backend routes file (`artifacts/api-server/src/routes/newsletter.ts`) likely has:
1. A `POST /api/admin/newsletter/generate` route that exists but uses an auth middleware checking for a token format that no longer matches what the frontend sends
2. No `POST` send route at all

### What to Do

**A. Fix the auth middleware on the generate route:**

Find the admin auth middleware being applied to the newsletter generate route. It likely checks:
```javascript
// Current broken check — looks for Bearer token from localStorage
const token = req.headers.authorization?.replace('Bearer ', '');
if (token !== process.env.ADMIN_PASSWORD) {
  return res.status(401).json({ error: 'Unauthorized' });
}
```

The problem is that `localStorage.getItem('afrienergy_admin_token')` is returning NULL. Either:
1. The admin login flow no longer sets `afrienergy_admin_token` in localStorage (if auth was refactored to session-based), OR
2. The token is being stored under a different key

**Fix option A (recommended) — make the newsletter admin routes use the same auth as the scraper admin routes:**

Check how the admin scraper routes (`/api/admin/projects/...`) authenticate. They clearly work because the admin can review/approve/reject projects. Copy that same auth mechanism to the newsletter routes:

```javascript
// Find how the scraper admin routes authenticate — likely something like:
// 1. Session cookie check
// 2. Or a different header/token pattern
// Whatever auth pattern the working admin routes use, apply it to newsletter routes too

// Example: if scraper routes use session auth
router.post('/api/admin/newsletter/generate', requireAdminSession, async (req, res) => { ... });
router.post('/api/admin/newsletter/send', requireAdminSession, async (req, res) => { ... });
```

**Fix option B — if the admin routes use localStorage token and it's just not being set:**

Find the admin login handler. After a successful admin login, ensure it returns the token:
```javascript
// In the admin login response
res.json({ success: true, token: process.env.ADMIN_PASSWORD });
```

And the frontend stores it:
```javascript
localStorage.setItem('afrienergy_admin_token', response.token);
```

**B. Add the missing send route:**

Add a POST send route that dispatches the newsletter:

```javascript
router.post('/api/admin/newsletter/send', adminAuth, async (req, res) => {
  try {
    const { newsletterId } = req.body;

    // Get the newsletter from DB
    const newsletter = await db.select()
      .from(newslettersTable)
      .where(eq(newslettersTable.id, newsletterId))
      .limit(1);

    if (!newsletter.length) {
      return res.status(404).json({ error: 'Newsletter not found' });
    }

    if (newsletter[0].status === 'sent') {
      return res.status(400).json({ error: 'Newsletter already sent' });
    }

    // Get subscribers
    const subscribers = await db.select()
      .from(subscribersTable)
      .where(eq(subscribersTable.active, true));

    // Send individually (NOT batch — batch returns 403 on current Resend plan)
    let sentCount = 0;
    for (const subscriber of subscribers) {
      try {
        await resend.emails.send({
          from: 'AfriEnergy Tracker <noreply@afrienergytracker.io>',
          to: subscriber.email,
          subject: newsletter[0].title,
          html: newsletter[0].contentHtml, // the rendered HTML email
        });
        sentCount++;
      } catch (err) {
        console.error(`Failed to send to ${subscriber.email}:`, err);
      }
    }

    // Update newsletter status
    await db.update(newslettersTable)
      .set({
        status: 'sent',
        sentAt: new Date(),
        recipientCount: sentCount
      })
      .where(eq(newslettersTable.id, newsletterId));

    res.json({ success: true, sentCount });
  } catch (error) {
    console.error('Newsletter send error:', error);
    res.status(500).json({ error: 'Failed to send newsletter' });
  }
});
```

**CRITICAL:** Use `resend.emails.send()` (individual), NOT `resend.emails.batch.send()`. The batch endpoint returns 403 on the current Resend plan. This was already diagnosed and documented in the earlier email delivery fix prompt.
