3. STRUCTURED EXTRACTION FORMAT — Every scraped/seeded project must conform to this schema before insert:

   {
     projectName: string (required),
     country: string (required — use standard country name, not ISO code),
     region: string (required — "North Africa" | "West Africa" | "East Africa" | "Central Africa" | "Southern Africa"),
     technology: string (required — "Solar" | "Wind" | "Hydro" | "Grid & Storage" | "Oil & Gas" | "Coal" | "Nuclear" | "Bioenergy"),
     status: string (required — "Operational" | "Under Construction" | "Development" | "Announced" | "Suspended" | "Active"),
     dealSizeUsdMn: number | null (in USD millions, e.g., 2000 for $2B),
     capacityMw: number | null,
     yearAnnounced: number | null,
     description: string (1-2 sentence summary),
     latitude: number | null,
     longitude: number | null,
     developer: string | null (lead developer/sponsor),
     investors: string | null (comma-separated financiers),
     newsUrl: string | null (source article URL),
     sourceUrl: string | null (primary data source URL)
   }

4. DEDUPLICATION LOGIC:
   - Before inserting any project, query existing DB for matches using:
     a) Exact match: same projectName + same country
     b) Fuzzy match: Levenshtein distance < 3 on projectName AND same country AND same technology
   - If match found: UPDATE existing record with any new non-null fields (merge, don't duplicate)
   - If no match: INSERT as new project
   - Log all decisions: "INSERTED: [name]", "UPDATED: [name] (matched existing ID [x])", "SKIPPED: [name] (duplicate)"
