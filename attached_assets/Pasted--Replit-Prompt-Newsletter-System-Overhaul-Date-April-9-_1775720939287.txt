## OVERVIEW — What's Changing and Why

The current newsletter system produces a single lengthy publication (~2,500-3,500 words, 9 sections). This needs to be restructured into **two distinct products** with different cadences, lengths, and purposes. Both must include **real visuals generated from the App's data** (charts, maps, tables) — not just text. The standalone "Global Context & External Intelligence" section must be removed; external intelligence should instead inform the analysis throughout.

## CHANGE 2: Create a New Biweekly "Africa Energy Brief"

### What
A new, shorter, denser publication called **"Africa Energy Brief"** — a condensed intelligence update published every two weeks. This is a quick-read briefing for busy professionals who want the key updates without the deep analysis.

### Content Strategy
The Brief should:
- Be **600-900 words** maximum (roughly 3-5 minute read)
- Draw from both the **App's database** (project count changes, new deals, stage progressions) and the **parent website Africa Energy Pulse** (news, policy updates)
- Be scannable: use bullet points, numbered lists, and short paragraphs
- NOT duplicate the monthly Insights — instead it fills the gap between monthly reports

### Brief Structure (5 sections, all short)

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
AFRICA ENERGY BRIEF
Biweekly Update — [Date]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. HEADLINE NUMBERS (2-3 lines)
   Key stats since last brief: new projects added, total
   investment change, notable milestones.

2. DEALS TO WATCH (3-5 bullet points)
   The most significant new or progressing deals — name,
   country, sector, deal size, what happened.

3. SECTOR PULSE (3-4 bullet points)
   Quick sector-level movements: which sectors gained
   projects, which saw financial closes, any notable shifts.

4. POLICY & MARKET SIGNALS (2-3 bullet points)
   Key external developments from Africa Energy Pulse
   and other sources that affect the investment landscape.
   This is where scraped external intelligence is woven in
   — attributed to source but NOT as a standalone dump.

5. ONE CHART (1 visual)
   A single data visualization from the App — could be
   sector breakdown, regional investment map, deal pipeline
   funnel, or month-over-month comparison.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Implementation

```javascript
// Biweekly: Every other Monday at 7:00 AM UTC
```

Both the monthly Insights and the biweekly Brief must include **data visualizations generated from the actual App data** — not just text. The current newsletter is pure text with no charts, maps, or visual tables. This is unacceptable.

### Types of Visuals to Generate

For the **monthly Insights** (include 3-5 visuals):
1. **Sector Investment Breakdown** — Horizontal bar chart showing total investment by sector (Solar, Wind, Hydro, Oil & Gas, etc.)
2. **Regional Investment Map** — A simplified map or table showing investment by region (East, West, North, Central, Southern Africa)
3. **Deal Pipeline Funnel** — Bar chart showing project counts by deal stage (Announced → Financial Close → Construction → Commissioned)

```typescript
async function generateChartViaAPI(config: object): Promise<string> {
```

```typescript
import puppeteer from 'puppeteer';

async function generateNewsletterPdf(newsletterId: number): Promise<Buffer> {
  const newsletter = await db.query('SELECT content_html, title FROM newsletters WHERE id = $1', [newsletterId]);
```

```javascript
import html2canvas from 'html2canvas';
import jsPDF from 'jspdf';

async function downloadPdf(elementId: string, title: string) {
  const element = document.getElementById(elementId);
  const canvas = await html2canvas(element, { scale: 2, useCORS: true });
  const imgData = canvas.toDataURL('image/png');
  const pdf = new jsPDF('p', 'mm', 'a4');
  const imgWidth = 190; // A4 width minus margins
  const imgHeight = (canvas.height * imgWidth) / canvas.width;

  let position = 10;
  let heightLeft = imgHeight;

  pdf.addImage(imgData, 'PNG', 10, position, imgWidth, imgHeight);
  heightLeft -= 277; // A4 page height minus margins

  while (heightLeft > 0) {
    position = heightLeft - imgHeight;
    pdf.addPage();
    pdf.addImage(imgData, 'PNG', 10, position, imgWidth, imgHeight);
    heightLeft -= 277;
  }

  pdf.save(`${title}.pdf`);
}
```
