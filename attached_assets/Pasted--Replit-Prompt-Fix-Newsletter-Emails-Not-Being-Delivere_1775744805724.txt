## ROOT CAUSE — Two Bugs

### Bug 1: Code Uses `resend.emails.batch.send()` Which Returns 403

The email dispatch code (likely in `artifacts/api-server/src/services/email-dispatch.ts` or inline in `artifacts/api-server/src/routes/newsletter.ts`) uses the Resend batch API:

```javascript
// BROKEN — returns 403 Forbidden on current Resend plan
await resend.emails.batch.send(emailsArray);
```

The Resend batch endpoint (`POST /emails/batch`) requires a higher-tier plan. On the current plan, it returns 403. The individual endpoint (`POST /emails`) works fine and returns 200.

### Bug 2: Newsletter Status Updated to "sent" Even When Send Fails

The code updates the newsletter status to `"sent"` and sets `recipientCount` to the total subscriber count BEFORE or REGARDLESS of whether emails were actually delivered. This means:
- The admin UI shows the newsletter as "sent" ✓
- The recipientCount shows 13 ✓
- But zero emails were actually sent ✗

**Replace with:**

## COMPLETE EXAMPLE: Fixed Send Function

Here's the complete send function to replace whatever currently exists:

```typescript
import { Resend } from 'resend';

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendNewsletter(
  newsletterId: number,
  db: any,
  newslettersTable: any,
  subscribersTable: any
) {
  // 1. Get the newsletter
  const [newsletter] = await db.select()
    .from(newslettersTable)
    .where(eq(newslettersTable.id, newsletterId))
    .limit(1);

  if (!newsletter) {
    throw new Error(`Newsletter ${newsletterId} not found`);
  }

  if (newsletter.status === 'sent') {
    throw new Error(`Newsletter ${newsletterId} already sent`);
  }

  // 2. Get active subscribers
  const subscribers = await db.select()
    .from(subscribersTable)
    .where(eq(subscribersTable.active, true));

  if (subscribers.length === 0) {
    throw new Error('No active subscribers found');
  }

  // 3. Send INDIVIDUALLY — do NOT use batch
  let sentCount = 0;
  const failedEmails: string[] = [];

  for (const subscriber of subscribers) {
    try {
      const result = await resend.emails.send({
        from: 'AfriEnergy Tracker <noreply@afrienergytracker.io>',
        to: subscriber.email,
        subject: newsletter.title,
        html: newsletter.contentHtml,
      });
      console.log(`✓ Sent to ${subscriber.email} — ID: ${result.data?.id}`);
      sentCount++;
    } catch (err: any) {
      console.error(`✗ Failed to send to ${subscriber.email}:`, err?.message || err);
      failedEmails.push(subscriber.email);
    }
  }

  // 4. Update status ONLY based on actual results
  if (sentCount > 0) {
    await db.update(newslettersTable)
      .set({
        status: 'sent',
        sentAt: new Date(),
        recipientCount: sentCount,
      })
      .where(eq(newslettersTable.id, newsletterId));

    console.log(`Newsletter ${newsletterId} sent to ${sentCount}/${subscribers.length} subscribers`);
    if (failedEmails.length > 0) {
      console.warn(`Failed recipients: ${failedEmails.join(', ')}`);
    }
  } else {
    console.error(`Newsletter ${newsletterId} FAILED — 0/${subscribers.length} sent`);
    // Keep as draft for retry
  }

  return { sentCount, totalSubscribers: subscribers.length, failedEmails };
}
```
