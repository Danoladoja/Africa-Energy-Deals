## STOP CONDITIONS

You **must** stop and ask before proceeding if any of the following are true:

1. A DFI source requires an API key or registration. Do not sign up on behalf of the project.
2. A DFI source blocks programmatic access or has no usable disclosure endpoint. Document what you found and ask whether to skip, defer, or use a scraping fallback.
3. Normalising a DFI's response requires a new field on `CandidateDraft` that doesn't exist yet. That's a signal the base shape is incomplete and we should add it deliberately, not ad-hoc.
4. Any adapter needs behaviour that the `BaseSourceAdapter` doesn't provide (new retry rules, a different auth header mechanism, etc.). That's a signal the base class needs extension — flag it and wait for a decision before forking behaviour inside the adapter.
