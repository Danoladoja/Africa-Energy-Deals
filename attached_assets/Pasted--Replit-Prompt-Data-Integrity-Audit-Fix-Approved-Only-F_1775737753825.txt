## CHANGE 1: Add Approved-Only Filtering to All Public API Endpoints (CRITICAL)

### Problem
The `GET /api/projects` endpoint (and any other public endpoints) returns all projects regardless of review status. Rejected, pending, and needs_source projects appear on the live site.

### File: `artifacts/api-server/src/routes/projects.ts` (or wherever project routes are defined)

### What to do:

**A. Add a default `reviewStatus = 'approved'` filter to the main GET /api/projects endpoint:**

Find the query that fetches projects for the public API. It currently does something like:

```javascript
// BROKEN — returns everything including rejected/pending projects
const projects = await db.query('SELECT * FROM projects ORDER BY created_at DESC LIMIT $1', [limit]);
```

Or if using Drizzle ORM:
```javascript
const projects = await db.select().from(projectsTable).orderBy(desc(projectsTable.createdAt)).limit(limit);
```

**Add a WHERE clause that filters by reviewStatus for all public (non-admin) requests:**

```javascript
// CORRECT — only return approved projects to public endpoints
const projects = await db.select()
  .from(projectsTable)
  .where(eq(projectsTable.reviewStatus, 'approved'))
  .orderBy(desc(projectsTable.createdAt))
  .limit(limit);
```

Or in raw SQL:
```sql
SELECT * FROM projects WHERE review_status = 'approved' ORDER BY created_at DESC LIMIT $1
```

The admin routes (e.g., `GET /api/admin/projects`, or any route behind admin auth) should continue to return ALL projects so the admin can review, approve, and reject. If admin routes share the same handler, add a parameter:

```javascript

  // ... query with WHERE review_status = 'approved'

// Admin route — all statuses, with optional filter

  const reviewStatus = req.query.reviewStatus; // optional filter for admin UI
  // ... query with optional WHERE clause
});
```

**C. Apply the same filter to ALL public endpoints that serve project data:**

Search the entire codebase for any endpoint that queries the `projects` table and returns data to the frontend. Common ones to check:

- Any endpoint that feeds data to the AI chatbot ← ADD FILTER
- The newsletter generator's data-fetching query ← ADD FILTER

**D. Update the AI chat/insights endpoint:**

### Root Cause
The application was built without a data access layer that distinguishes between "all data" (admin) and "public data" (approved only). The `reviewStatus` field exists and the admin review queue works, but the approval gate was never enforced on the API endpoints that serve data to the public frontend. Every page consumes the same unfiltered `GET /api/projects` endpoint.
