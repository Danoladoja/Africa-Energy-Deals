# Prompt: Integrate 9 Backend Scraper Improvement Files

## Instructions

I have 9 files — 5 brand new and 4 modified — that implement backend scraper improvements. These were written externally to match the existing codebase patterns exactly. Your job is to place them into the project at the correct paths. **Do NOT modify the file contents.** Just integrate them.

## New files (create these)

### 1. `artifacts/api-server/src/services/deal-signal-filter.ts`
Keyword gate that runs before LLM calls — filters out non-deal articles (opinion, policy, etc.) to save LLM cost. Exports `isDealSignal(text, adapter?)`.

### 2. `artifacts/api-server/src/services/article-fetcher.ts`
Fetches full article HTML from a URL, strips it to readable text, and returns it for use in the LLM extraction prompt. Exports `fetchArticleText(url)`.

### 3. `artifacts/api-server/src/services/batch-dedup.ts`
Deduplicates candidates within a single scraper batch by normalized name + country. Keeps the candidate with the highest completeness score and merges unique fields from duplicates. Exports `deduplicateBatch(candidates)` and `deduplicateBatchByUrl(candidates)`.

### 4. `artifacts/api-server/src/services/adapter-trust.ts`
Tracks per-adapter historical accuracy (approval/review/rejection rates) and computes a dynamic trust factor (0.6–1.2) that adjusts adapter confidence in the routing engine. Reads from existing `scraper_runs` table — no new tables needed. Exports `getAdapterTrustFactor(key)`, `getAllTrustScores()`, `refreshTrustScores()`.

### 5. `artifacts/api-server/src/scraper/adapters/api-worldbank.ts`
World Bank Projects API adapter. Pulls structured energy project data from Africa. Returns pre-structured JSON with confidence 1.0 — no LLM extraction needed. Exports `worldBankApiAdapter`.
