### 1. Fix the Sitemap — Create a Dynamic Sitemap Generator

**This is the #1 priority.** Without a working sitemap, Google cannot efficiently discover the site's pages.

**File:** Add a server-side route that generates `sitemap.xml` dynamically from the database.

```
GET /sitemap.xml → Returns XML content-type with a valid sitemap
```

**Implementation:**

```javascript
// In railway-server.mjs or a route file
app.get('/sitemap.xml', async (req, res) => {
  const baseUrl = 'https://afrienergytracker.io';

  // Static pages
  const staticPages = [
    { url: '/', changefreq: 'daily', priority: '1.0' },
    { url: '/dashboard', changefreq: 'daily', priority: '0.9' },
    { url: '/deals', changefreq: 'daily', priority: '0.9' },
    { url: '/map', changefreq: 'weekly', priority: '0.8' },
    { url: '/countries', changefreq: 'weekly', priority: '0.8' },
    { url: '/developers', changefreq: 'weekly', priority: '0.7' },
    { url: '/studio', changefreq: 'monthly', priority: '0.5' },
    { url: '/api-docs', changefreq: 'monthly', priority: '0.4' },
  ];

  // Dynamic pages from database
  const projects = await db.query('SELECT id, "projectName", "createdAt" FROM projects');
  const countries = await db.query('SELECT DISTINCT country FROM projects WHERE country IS NOT NULL');
  const developers = await db.query('SELECT DISTINCT developer FROM projects WHERE developer IS NOT NULL');

  // Build XML
  let xml = '<?xml version="1.0" encoding="UTF-8"?>\n';
  xml += '<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">\n';

  // Static pages
  for (const page of staticPages) {
    xml += `  <url>\n`;
    xml += `    <loc>${baseUrl}${page.url}</loc>\n`;
    xml += `    <changefreq>${page.changefreq}</changefreq>\n`;
    xml += `    <priority>${page.priority}</priority>\n`;
    xml += `  </url>\n`;
  }

  // Deal detail pages
  for (const project of projects) {
    xml += `  <url>\n`;
    xml += `    <loc>${baseUrl}/deals/${project.id}</loc>\n`;
    xml += `    <changefreq>weekly</changefreq>\n`;
    xml += `    <priority>0.7</priority>\n`;
    if (project.createdAt) {
      xml += `    <lastmod>${new Date(project.createdAt).toISOString().split('T')[0]}</lastmod>\n`;
    }
    xml += `  </url>\n`;
  }

  // Country profile pages
  for (const row of countries) {
    const slug = row.country.toLowerCase().replace(/\s+/g, '-');
    xml += `  <url>\n`;
    xml += `    <loc>${baseUrl}/countries/${slug}</loc>\n`;
    xml += `    <changefreq>weekly</changefreq>\n`;
    xml += `    <priority>0.7</priority>\n`;
    xml += `  </url>\n`;
  }

  // Developer profile pages
  for (const row of developers) {
    const slug = row.developer.toLowerCase().replace(/\s+/g, '-');
    xml += `  <url>\n`;
    xml += `    <loc>${baseUrl}/developers/${slug}</loc>\n`;
    xml += `    <changefreq>monthly</changefreq>\n`;
    xml += `    <priority>0.5</priority>\n`;
    xml += `  </url>\n`;
  }

  xml += '</urlset>';

  res.set('Content-Type', 'application/xml');
  res.send(xml);
});
```

**IMPORTANT:** This server-side route MUST be registered BEFORE the SPA catch-all route. Otherwise the catch-all will intercept `/sitemap.xml` and return the HTML shell (which is the current bug).

### 13. Server-Side Rendering (SSR) Consideration for Crawlers

Since this is a client-side React SPA, search engine crawlers see the initial HTML shell before JavaScript renders the content. While Googlebot can execute JavaScript, other crawlers (Bing, social media previews) may not.

**Recommended:** Add a lightweight server-side meta tag injection for crawlers. In `railway-server.mjs`, before serving the SPA HTML for any route:

```javascript
// For bot requests, inject meta tags into the HTML shell
app.get('*', async (req, res, next) => {
  const userAgent = req.headers['user-agent'] || '';
  const isBot = /googlebot|bingbot|slurp|duckduckbot|facebookexternalhit|twitterbot|linkedinbot|whatsapp/i.test(userAgent);

  if (isBot) {
    // Read the index.html template
    let html = fs.readFileSync(path.join(distPath, 'index.html'), 'utf-8');

    // Generate meta tags based on the URL path
    const meta = await generateMetaForPath(req.path);

    // Inject into <head>
    html = html.replace('</head>',
      `<title>${meta.title}</title>
       <meta name="description" content="${meta.description}" />
       <meta property="og:title" content="${meta.title}" />
       <meta property="og:description" content="${meta.description}" />
       <meta property="og:url" content="https://afrienergytracker.io${req.path}" />
       <meta property="og:image" content="${meta.ogImage || 'https://afrienergytracker.io/og-image.png'}" />
       <link rel="canonical" href="https://afrienergytracker.io${req.path}" />
       ${meta.jsonLd ? `<script type="application/ld+json">${JSON.stringify(meta.jsonLd)}</script>` : ''}
       </head>`
    );

    return res.send(html);
  }

  next(); // For non-bot requests, serve the SPA normally
});
```

The `generateMetaForPath()` function should:
- Parse the URL path (e.g., `/countries/egypt` → lookup Egypt data)
- Query the database for dynamic pages (deal details, country profiles, developer profiles)
- Return title, description, OG tags, and JSON-LD appropriate for that page
- Fall back to generic site-level metadata for unknown paths

This ensures that when Google, Facebook, Twitter, LinkedIn, or WhatsApp crawl any URL, they receive proper metadata in the initial HTML — even before JavaScript executes.
