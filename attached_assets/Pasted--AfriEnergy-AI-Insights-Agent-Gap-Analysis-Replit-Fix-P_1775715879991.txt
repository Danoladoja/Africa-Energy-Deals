**Observation:** The `/insights` page causes browser timeouts and fails to load. The page hangs indefinitely, suggesting either an infinite loop, a blocking API call that never resolves, or a rendering crash in the 739-line `insights.tsx` component.

**File:** `artifacts/energy-tracker/src/pages/insights.tsx`

**What to investigate and fix:**
- Check for any `useEffect` or `useQuery` hooks that fire on mount and never resolve (e.g., fetching from an endpoint that errors silently)
- Check if the page depends on an API endpoint (`/api/chat` or similar) that blocks rendering when it fails
- Check for infinite re-render loops caused by state updates inside effects
- Add error boundaries and loading states so the page renders even if API calls fail
- The page should show the empty state (greeting + insight cards) immediately without waiting for any API call

**Priority:** 🔴 CRITICAL — The entire Insights feature is unusable if the page doesn't load.
