## WHY WE'RE MIGRATING

The Resend free tier has a **5 emails/day** and **41/month** quota. The newsletter has 13 subscribers, so a single send exhausts the daily limit. Brevo's free tier allows **300 emails/day** (9,000/month), which is more than enough.

The Resend API itself works — we confirmed a successful test send returned `{"data":{"id":"bf87a390..."}}`. The problem is purely the quota.

## STEP 3: Replace Resend with Brevo

### Old Resend code (find and replace):

```typescript
import { Resend } from 'resend';
const resend = new Resend(process.env.RESEND_API_KEY);

// Sending an email:
const result = await resend.emails.send({
  from: 'AfriEnergy Tracker <noreply@afrienergytracker.io>',
  to: subscriber.email,
  subject: newsletter.title,
  html: newsletter.contentHtml,
});
```

### New Brevo code:

```typescript
import { BrevoClient } from '@getbrevo/brevo';

const brevo = new BrevoClient({
  apiKey: process.env.BREVO_API_KEY!,
});

// Sending an email:
const result = await brevo.transactionalEmails.sendTransacEmail({
  sender: {
    name: 'AfriEnergy Tracker',
    email: 'noreply@afrienergytracker.io',
  },
  to: [{ email: subscriber.email }],
  subject: newsletter.title,
  htmlContent: newsletter.contentHtml,
});
```

### Key differences:
| Resend | Brevo |
|---|---|
| `from: 'Name <email>'` | `sender: { name: '...', email: '...' }` |
| `to: 'email@...'` | `to: [{ email: '...' }]` (array of objects) |
| `html: '...'` | `htmlContent: '...'` |
| `resend.emails.send({...})` | `brevo.transactionalEmails.sendTransacEmail({...})` |
| `RESEND_API_KEY` env var | `BREVO_API_KEY` env var |
| `result.data.id` | `result.messageId` |

## STEP 5: Update the Test Email Endpoint

Update the existing `/api/test-email` endpoint (or `/api/admin/newsletter/test-email`) to use Brevo instead of Resend:

```typescript
app.get('/api/test-email', async (req, res) => {
  const diagnostics: any = { timestamp: new Date().toISOString(), steps: [] };

  try {
    const apiKey = process.env.BREVO_API_KEY;
    diagnostics.steps.push({
      step: 'env_check',
      keyExists: !!apiKey,
      keyPrefix: apiKey ? apiKey.substring(0, 8) + '...' : 'NOT SET',
    });

    if (!apiKey) {
      diagnostics.error = 'BREVO_API_KEY is not set';
      return res.json(diagnostics);
    }

    const { BrevoClient } = require('@getbrevo/brevo');
    const brevo = new BrevoClient({ apiKey });

    const result = await brevo.transactionalEmails.sendTransacEmail({
      sender: { name: 'AfriEnergy Tracker', email: 'noreply@afrienergytracker.io' },
      to: [{ email: 'danoladoja@gmail.com' }],
      subject: `AfriEnergy Test — ${new Date().toLocaleTimeString()}`,
      htmlContent: '<h2>Test Email from Brevo</h2><p>If you see this, Brevo is working. Sent at: ' + new Date().toISOString() + '</p>',
    });

    diagnostics.steps.push({ step: 'send_complete', result });
    diagnostics.success = true;
    return res.json(diagnostics);
  } catch (err: any) {
    diagnostics.steps.push({ step: 'error', message: err?.message, statusCode: err?.statusCode });
    diagnostics.success = false;
    diagnostics.error = err?.message || String(err);
    return res.json(diagnostics);
  }
});
```

## STEP 7: Handle the Brevo SDK Version

The Brevo Node.js SDK has two versions. If `@getbrevo/brevo` v5+ uses the `BrevoClient` import shown above, great. If the installed version is v3.x/v4.x, the import pattern is different:

### v3.x/v4.x fallback (if BrevoClient import fails):

```typescript
import * as Brevo from '@getbrevo/brevo';

const apiInstance = new Brevo.TransactionalEmailsApi();
apiInstance.setApiKey(
  Brevo.TransactionalEmailsApiApiKeys.apiKey,
  process.env.BREVO_API_KEY!
);

// Sending:
const sendSmtpEmail = new Brevo.SendSmtpEmail();
sendSmtpEmail.sender = { name: 'AfriEnergy Tracker', email: 'noreply@afrienergytracker.io' };
sendSmtpEmail.to = [{ email: subscriber.email }];
sendSmtpEmail.subject = newsletter.title;
sendSmtpEmail.htmlContent = contentHtml;

const result = await apiInstance.sendTransacEmail(sendSmtpEmail);
console.log(`✓ Sent — messageId: ${result.body.messageId}`);
```

**Try v5 first (BrevoClient import). If it fails to compile, use the v3/v4 pattern above.**

### Getting the Brevo API Key:
1. Go to https://app.brevo.com
2. Sign up for a free account (no credit card needed)
3. Go to SMTP & API → API Keys
4. Create a new API key
5. The key starts with `xkeysib-`

## DOMAIN VERIFICATION IN BREVO

For emails to send from `noreply@afrienergytracker.io`, the domain must be verified in Brevo:

1. Go to Brevo dashboard → Settings → Senders, Domains & Dedicated IPs
2. Click "Add a domain"
3. Enter `afrienergytracker.io`
4. Brevo will give you DNS records (TXT/CNAME) to add
5. Add these records in the Hostinger DNS panel (afrienergytracker.io DNS is managed at Hostinger)
6. Come back to Brevo and click "Verify"

**IMPORTANT:** Until the domain is verified, Brevo may still send emails using their shared sending infrastructure, but deliverability will be lower. Verify the domain for production use.

**NOTE:** If domain verification blocks immediate testing, Brevo allows sending from verified sender email addresses. Daniel can add `danoladoja@gmail.com` as a verified sender in Brevo → Settings → Senders to test immediately.
