## Task 6: Fix the Review Auth Bug (Backend)

In `artifacts/api-server/src/middleware/reviewAuth.ts` (or wherever the review authentication middleware lives):

- **Find:** The role-checking logic that gates `/api/review/*` endpoints
- **Bug:** The user's role is stored as `"ADMIN"` (uppercase) in the database, but the middleware checks for lowercase `"admin"` or `"reviewer"`. This causes 403 "Reviewer access required" errors for admin users.
- **Fix:** Normalize the role comparison to be case-insensitive:
```typescript
// Before (broken):
if (user.role !== 'admin' && user.role !== 'reviewer') {
  return res.status(403).json({ error: 'Reviewer access required' });
}

// After (fixed):
const role = (user.role || '').toUpperCase();
if (role !== 'ADMIN' && role !== 'REVIEWER') {
  return res.status(403).json({ error: 'Reviewer access required' });
}
```
