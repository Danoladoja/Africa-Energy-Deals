## TASK: Add a Diagnostic Test Email Endpoint

Add a **temporary** diagnostic endpoint that sends a single test email and returns the **full Resend API response** (or error) in the HTTP response body. This bypasses the newsletter flow entirely to isolate whether Resend itself is working.

### Add this route in `artifacts/api-server/src/routes/newsletter.ts` (or wherever newsletter routes are registered):

```typescript
// TEMPORARY DIAGNOSTIC — remove after confirming email works
router.get('/api/admin/newsletter/test-email', async (req, res) => {
  try {
    // Step 1: Check if RESEND_API_KEY is set
    const apiKey = process.env.RESEND_API_KEY;
    if (!apiKey) {
      return res.json({
        success: false,
        error: 'RESEND_API_KEY environment variable is not set',
        step: 'env_check'
      });
    }

    // Log key prefix for verification (first 8 chars only — safe to log)
    const keyPrefix = apiKey.substring(0, 8);
    console.log(`[TEST-EMAIL] RESEND_API_KEY starts with: ${keyPrefix}...`);

    // Step 2: Initialize Resend and send one test email
    const { Resend } = await import('resend');
    const resend = new Resend(apiKey);

    console.log('[TEST-EMAIL] Sending test email to danoladoja@gmail.com...');

    const result = await resend.emails.send({
      from: 'AfriEnergy Tracker <noreply@afrienergytracker.io>',
      to: 'danoladoja@gmail.com',
      subject: 'AfriEnergy Test Email — Diagnostic Check',
      html: `
        <div style="font-family: Arial, sans-serif; max-width: 600px; margin: 0 auto; padding: 20px;">
          <h2 style="color: #1a5632;">AfriEnergy Tracker — Test Email</h2>
          <p>This is a diagnostic test email sent at <strong>${new Date().toISOString()}</strong>.</p>
          <p>If you received this, the Resend API is working correctly with your domain.</p>
          <hr style="border: 1px solid #e0e0e0;">
          <p style="color: #888; font-size: 12px;">Sent from afrienergytracker.io via Resend</p>
        </div>
      `,
    });

    console.log('[TEST-EMAIL] Resend response:', JSON.stringify(result));

    // Step 3: Return full response
    return res.json({
      success: true,
      resendResponse: result,
      keyPrefix: keyPrefix + '...',
      timestamp: new Date().toISOString(),
      step: 'send_complete'
    });

  } catch (err: any) {
    console.error('[TEST-EMAIL] Error:', err);
    return res.json({
      success: false,
      error: err?.message || String(err),
      errorName: err?.name,
      statusCode: err?.statusCode,
      fullError: JSON.stringify(err, Object.getOwnPropertyNames(err)),
      step: 'send_failed'
    });
  }
});
```

## HOW TO USE THE DIAGNOSTIC

After deploying, Daniel should:

1. Navigate to `https://afrienergytracker.io/api/admin/newsletter/test-email` while logged into the admin panel
2. The page will show a JSON response with one of these outcomes:

**If RESEND_API_KEY is missing:**
```json
{ "success": false, "error": "RESEND_API_KEY environment variable is not set", "step": "env_check" }
```
→ Fix: Add `RESEND_API_KEY` in Railway environment variables

**If Resend API rejects the request:**
```json
{ "success": false, "error": "...", "statusCode": 403, "step": "send_failed" }
```
→ Check the error message — could be invalid API key, unverified domain, or plan limits

**If Resend API accepts:**
```json
{ "success": true, "resendResponse": { "data": { "id": "abc123..." } }, "step": "send_complete" }
```
→ The API is working; check spam folder. If `data.id` is present, Resend accepted the email.

3. Check the Railway logs for the `[TEST-EMAIL]` lines
4. Check your inbox (and spam folder) for the test email
